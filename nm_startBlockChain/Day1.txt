How TypeScript Works?

- 코드를 실행하기 전에 에러를 잡기 위해 타입스크립트를 사용한다
- strongly typed 언어이다
- 작성한 코드가 자바스크립트로 변환된다
  브라우저가 ts가 아닌 js를 이해하기 때문이다
  node.js는 둘 다 이해 할 수 있다
- typescript는 개발자의 실수를 막아준다
  어떻게?
  컴파일은 그저 작성한 ts 코드를 js 코드로 바꿔준다
  ts가 제공하는 보호장치는 ts 코드가 js로 변환되기 전에 발생한다
  ts가 먼저 우리의 코드를 확인해서 변환된 js안에서 실수가 일어나지 않게 막는다

정리하자면 내가 ts코드를 쓰고 컴파일하면 보호장치 없는 아주 정상적인 JS코드가 된다
but ts 코드에 에러가 있다면 js코드로 변환되지 않는다
ts가 에러가 발생할 거 같은 코드를 감지하면 컴파일 되지 않는다

TS는 JS를 이해하고 타입 추론(type inference)을 사용하여
추가 코드 없이도 훌륭한 도구를 제공한다

TS의 두 가지 접근 방식
1. 데이터와 변수의 타입을 명시적으로 정의
2. JS처럼 변수만 생성하고 넘어가기(TS가 타입을 추론해줌)
